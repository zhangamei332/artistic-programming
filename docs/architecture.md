## 2. 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端框架 | React 18 + TypeScript | 生态丰富，社区成熟 |
| 构建工具 | Vite | 快速开发，HMR 即时热更新 |
| 状态管理 | Zustand | 轻量，无模板代码 |
| 节点画布 | React Flow | 成熟的节点编辑器库 |
| 代码编辑器 | Monaco Editor | VS Code 同款内核 |
| UI 组件库 | Ant Design | 中文友好，组件齐全 |
| 样式方案 | CSS Modules + 全局变量 | 简单可控 |
| 后端框架 | Express + TypeScript | 轻量灵活 |
| ORM | Prisma | 类型安全，迁移方便 |
| 数据库 | PostgreSQL | 成熟稳定，JSON 支持好 |
| AI SDK | 各模型官方 SDK | Gemini / OpenAI / DeepSeek |
| 认证 | JWT + bcrypt | 无状态认证 |

## 3. 核心数据流

### 3.1 文生代码流程

```
用户输入文字 → 前端 POST /api/generate/text
  → 后端 AI Service 调用对应模型
  → 模型返回代码
  → 后端 Code Parser 解析代码为节点
  → 返回 { code, nodes, language } 给前端
  → 前端更新代码编辑器 + 节点画布 + 预览
```

### 3.2 图生代码流程

```
用户上传图片 → 前端 POST /api/generate/image (multipart)
  → 后端保存图片 + 调用 AI Vision API 分析
  → AI 生成对应风格代码
  → 后端 Code Parser 解析代码为节点
  → 返回 { code, nodes, language } 给前端
```

### 3.3 节点参数修改流程

```
用户修改节点参数 → 前端更新节点状态
  → 前端重组代码（根据节点参数重写变量值）
  → 前端更新代码编辑器 + 预览刷新
```

### 3.4 代码编辑反向同步流程

```
用户编辑代码 → 前端检测代码变化
  → 前端 POST /api/parse/code
  → 后端 Code Parser 重新解析
  → 返回新节点结构
  → 前端更新节点画布
```

## 4. API 设计原则

- RESTful 风格
- JSON 格式通信
- 所有 AI 相关接口需认证
- 错误统一返回格式：`{ error: string, code: number }`
- 分页列表统一格式：`{ data: T[], total: number, page: number }`

---

## 5. 安全设计

- API Key 仅存储后端环境变量，绝不返回前端
- 用户密码 bcrypt 加盐哈希
- JWT token 过期时间 7 天
- 预览 iframe 使用 sandbox 属性隔离
- 代码执行限制在 iframe sandbox 中，禁止网络请求
- 上传文件限制：图片 ≤ 10MB，仅允许 jpg/png/webp
- API 限流：每用户每分钟最多 10 次 AI 生成请求
